CASE STUDY

1. Project Overview

Hybrid Identity Transformation: From Active Directory to Modern Endpoint Management

Business Scenario: SweetCrust Bakery Ltd — 20 Employee Small Business

Project Type: Hybrid Identity & Endpoint Modernization

Role Alignment: Systems Administrator | IAM Analyst | Microsoft 365 Administrator | Endpoint Administrator

This case study demonstrates how a small business can be managed using the same identity, access, endpoint, and cloud management technologies commonly used in medium and large enterprise environments.

SweetCrust Bakery Ltd represents a 20-employee pastry business operating across production, sales, finance, and management functions. The business required centralized identity management, secure cloud integration, modern endpoint management, and a scalable foundation for Microsoft 365 adoption.

Although the scenario is based on a small business, the implementation pattern reflects an enterprise-ready architecture: Active Directory Domain Services for on-premises identity, Microsoft Entra Connect for hybrid synchronization, Microsoft Entra ID for cloud identity, and Microsoft Intune for modern endpoint management.

The goal of this project was to demonstrate a realistic transformation journey from traditional on-premises identity management to a cloud-integrated Microsoft ecosystem.

3. The Business Problem

SweetCrust Bakery Ltd required a centralized and scalable way to manage employee identities, Windows devices, access control, and Microsoft 365 services.

As the business expanded, manual user creation, inconsistent device setup, and lack of centralized security controls created operational and security challenges.

  • Employee accounts needed to be managed consistently.
  • Windows devices required centralized authentication and policy control.
  • The business needed a path from on-premises identity to cloud identity.
  • Remote and hybrid work required cloud-based endpoint management.
  • Access to Microsoft 365 services needed to be secured and governed.

4. Solution Architecture & Design

The solution was designed as a staged transformation: first establishing an on-premises Active Directory environment, then extending identity into Microsoft Entra ID, and finally managing endpoints through Microsoft Intune.

SweetCrust Bakery Ltd 20-employee business environment
↓
Windows Server Core on-premises infrastructure
↓
Active Directory Domain Services Users, groups, DNS, authentication, Group Policy
↓
Microsoft Entra Connect Sync Synchronizes on-premises identities to the cloud
↓
Microsoft Entra ID Cloud identity, Microsoft 365 access, Conditional Access
↓
Microsoft Intune Endpoint enrollment, compliance, app deployment, device security
↓
Managed Windows 10 / Windows 11 Endpoints Corporate devices managed from the cloud

This architecture mirrors how many small and medium-sized organizations modernize their identity and endpoint environment without immediately abandoning existing on-premises infrastructure.

5. Implementation Phases

Phase 1: On-Premises Identity Foundation

Built the initial Active Directory environment using Windows Server, configured DNS, created users and groups, joined a Windows client to the domain, and validated Group Policy.

View Phase 1 Detailed Implementation

Phase 2: Hybrid Directory Synchronization

Extended the on-premises Active Directory environment into Microsoft Entra ID using Microsoft Entra Connect Sync, enabling synchronized identities and hybrid cloud access.

View Phase 2 Detailed Implementation

Phase 3: Modern Endpoint Management

Transitioned device management into the cloud using Microsoft Intune, including Windows enrollment, Autopilot, compliance policies, Conditional Access, application deployment, and endpoint security controls.

  • Confirmed Windows client domain join using delegated credentials.
  • Validated user authentication against Active Directory.
  • Verified Group Policy application using gpupdate and gpresult.
  • Confirmed user synchronization into Microsoft Entra ID.
  • Verified hybrid join state using dsregcmd /status.
  • Validated Intune enrollment and endpoint compliance reporting.
  • Tested Conditional Access enforcement against device compliance state.

8. Engineering Takeaways

  • Hybrid identity provides a practical migration path for organizations moving from traditional Active Directory to Microsoft cloud services.
  • Active Directory remains important in many environments, but Entra ID and Intune enable modern identity and endpoint management beyond the local network.
  • Group Policy and Intune configuration profiles solve similar problems, but Intune extends management to cloud-connected and remote devices.
  • Conditional Access becomes more powerful when identity, device compliance, and endpoint management are connected.
  • A phased transformation approach reduces risk by modernizing identity and endpoint management without immediately discarding existing infrastructure.
